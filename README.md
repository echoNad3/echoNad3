### 🛠️ I make stuff I want to use

Most of what I build starts the same way: **I need a tool, I look for one**, and everything I find is either full of ads, locked behind a subscription, collecting more data than it needs, or bloated with features nobody asked for.

**So I make my own.**

That has turned into fitness apps, VS Code extensions, YouTube summarizers, small utilities, automation tools, and a few video game projects.

---

### 💭 Why

I’m tired of simple software being turned into a service.

You install something to do one basic job, and suddenly it wants an account, cloud access, analytics, notifications, a monthly payment, and permission to track half your life.

A lot of good software also gets worse over time because the company behind it needs more growth, more engagement, and more ways to squeeze money out of the same users.

**I don’t want any of that in the things I use.**

---

### 🎯 What I aim for

* **useful**
* **simple**
* **fast**
* **private where possible**
* **no ads**
* **no pointless accounts**
* **no subscriptions for basic features**

Some projects are public. Some are unfinished. A lot of them are just things I built because I wanted them to exist.


**That’s pretty much it.**
